Analyze our conversation and generate spaced-repetition flashcards capturing the key things I learned in this session.

## Critical Rules

1. **NEVER edit ~/.claude-flashcards/cards.json directly.** Always use the script commands.
2. **ALWAYS use the script to save cards** -- the script handles ID generation, SM-2 scheduling, and data integrity.
3. Only provide `question`, `answer`, `category`, `source_repo`, and optionally `topic` when generating cards. All other fields are auto-generated.

## What to Extract

Good flashcard candidates from a coding session:

- **Conceptual insights**: "Why does X work this way?" -- understanding that transfers across projects
- **Debugging patterns**: "When you see error X, check Y" -- diagnostic knowledge that saves future time
- **Tool/API usage**: "How to use X to accomplish Y" -- practical skills easy to forget
- **Architecture decisions**: "Why pattern X over Y here" -- design reasoning
- **Language/framework gotchas**: Non-obvious behavior, edge cases, idiomatic patterns

## What NOT to Extract

- File paths, variable names, or details specific to one repo
- Boilerplate or syntax easily found with a web search
- Trivial facts the user already clearly knew
- Implementation details that only matter for one task

## Card Quality Rules

1. **One concept per card** -- never combine multiple ideas
2. **Question tests understanding, not recall** -- "Why does..." beats "What is the command for..."
3. **Answer is concise but complete** -- 1-3 sentences, optionally with a short code snippet
4. **Include code only when it's the point** -- a 2-line example, not a 20-line function
5. **Ground in the user's context** -- reference the problem they were solving

## Categories

Assign exactly one per card:

- `concept` -- theoretical understanding, mental models
- `pattern` -- reusable code or architecture patterns
- `debugging` -- diagnostic knowledge, error resolution
- `tool` -- CLI tools, APIs, framework usage
- `architecture` -- system design decisions and trade-offs

## Topic Field

Optionally assign a `topic` string to group related cards across sessions (e.g., "Claude Agent SDK", "React hooks", "Docker networking"). Before creating a new topic, check existing cards to reuse an existing topic name if one fits. Keep topics short (2-4 words). Do NOT add `theme`, `tags`, or other fields not in the schema below.

## Card JSON Format

When generating cards, provide ONLY these fields:

```json
[
  {
    "question": "Why does...",
    "answer": "Because...",
    "category": "concept",
    "source_repo": "repo-name",
    "topic": "Optional Topic Name"
  }
]
```

The `id`, `created`, and SM-2 fields are auto-generated by the script. Do NOT include them.

## Steps

1. Determine the source repo by running:
   `git rev-parse --show-toplevel`
   and using the basename of the result as `source_repo`. If it fails, use "unknown".

2. Check existing topics for consistency by running:
   `python3 ${CLAUDE_PLUGIN_ROOT}/scripts/flashcards list-due`
   and extracting unique topic names from the output to reuse where appropriate.

3. Review the full conversation for learning moments
4. Generate 3-8 high-quality cards (prefer fewer, better cards)
5. For each card provide ONLY: question, answer, category, source_repo, and optionally topic
6. Do NOT include id, created, sm2, theme, tags, or any other fields
7. Reuse existing topic names when a card fits an existing topic
8. Present the cards in a numbered list so I can review them
9. If I ask to remove or edit any cards, do that before saving

### Presentation Format

```
Here are N cards from this session:

1. [concept] Why does Python's context manager __exit__ receive exception info even on success?
   -> Because __exit__ always receives (exc_type, exc_val, exc_tb), all None when no exception.
      This lets one method handle both cleanup and error suppression.

2. [debugging] What does Docker exit code 137 indicate?
   -> The process was killed by SIGKILL, usually from OOM. Check container memory limits.
```

10. Save by piping the JSON array to the script (NEVER edit cards.json directly):
    `echo '<json_array>' | python3 ${CLAUDE_PLUGIN_ROOT}/scripts/flashcards add`
11. Show a brief summary: count, categories, topics

## Error Handling

- **No cards exist**: "No flashcards yet. Run /flashcards after a coding session to generate your first cards."
- **Script not found**: "The flashcards script wasn't found. The plugin may need to be reinstalled."

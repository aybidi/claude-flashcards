# Session Flashcards

Generate high-quality spaced-repetition flashcards from Claude Code sessions and review them with SM-2 scheduling.

## Card Generation

### What to extract

Good flashcard candidates from a coding session:

- **Conceptual insights**: "Why does X work this way?" -- understanding that transfers across projects
- **Debugging patterns**: "When you see error X, check Y" -- diagnostic knowledge that saves future time
- **Tool/API usage**: "How to use X to accomplish Y" -- practical skills easy to forget
- **Architecture decisions**: "Why pattern X over Y here" -- design reasoning
- **Language/framework gotchas**: Non-obvious behavior, edge cases, idiomatic patterns

### What NOT to extract

- File paths, variable names, or details specific to one repo
- Boilerplate or syntax easily found with a web search
- Trivial facts the user already clearly knew
- Implementation details that only matter for one task

### Card quality rules

1. **One concept per card** -- never combine multiple ideas
2. **Question tests understanding, not recall** -- "Why does..." beats "What is the command for..."
3. **Answer is concise but complete** -- 1-3 sentences, optionally with a short code snippet
4. **Include code only when it's the point** -- a 2-line example, not a 20-line function
5. **Ground in the user's context** -- reference the problem they were solving

### Categories

Assign exactly one per card:

- `concept` -- theoretical understanding, mental models
- `pattern` -- reusable code or architecture patterns
- `debugging` -- diagnostic knowledge, error resolution
- `tool` -- CLI tools, APIs, framework usage
- `architecture` -- system design decisions and trade-offs

### Card JSON format

When generating cards, provide these fields:

```json
[
  {
    "question": "Why does...",
    "answer": "Because...",
    "category": "concept",
    "source_repo": "repo-name"
  }
]
```

The `id`, `created`, and SM-2 fields are auto-generated by the script.

## Generating Cards (via /flashcards)

1. Review the conversation for learning moments
2. Generate 3-8 cards (prefer fewer, higher-quality cards over many weak ones)
3. Present cards in a numbered list for the user to see
4. Save by piping JSON array to the script:

```bash
echo '[{...}]' | python3 ${CLAUDE_PLUGIN_ROOT}/scripts/flashcards add
```

### Presentation format

```
Here are N cards from this session:

1. [concept] Why does Python's context manager __exit__ receive exception info even on success?
   -> Because __exit__ always receives (exc_type, exc_val, exc_tb), all None when no exception.
      This lets one method handle both cleanup and error suppression.

2. [debugging] What does Docker exit code 137 indicate?
   -> The process was killed by SIGKILL, usually from OOM. Check container memory limits.
```

Save the cards, then show a summary: "Saved N cards (breakdown by category)."

If the user asks to remove or edit a card after seeing the list, do so before saving.

## Reviewing Cards (via /review)

1. Load due cards:
```bash
python3 ${CLAUDE_PLUGIN_ROOT}/scripts/flashcards list-due
```

2. If no cards are due, say so and show stats from:
```bash
python3 ${CLAUDE_PLUGIN_ROOT}/scripts/flashcards stats
```

3. Present cards one at a time -- question first, then wait for the user.

4. After the user says "show" or similar, reveal the answer.

5. Ask for a rating: (1) Again  (2) Hard  (3) Good  (4) Easy

6. Record it:
```bash
python3 ${CLAUDE_PLUGIN_ROOT}/scripts/flashcards rate <card_id> <0-3>
```

7. If the user rates "Again" (1), briefly re-explain the concept to reinforce it.

8. After all cards, show a summary of the session.

### Review presentation

```
Card 1/5  |  category  |  source-repo

Q: [question]

Say "show" to reveal the answer.
```

After reveal:

```
A: [answer]

How did you do?  (1) Again  (2) Hard  (3) Good  (4) Easy
```

## Error Handling

- **No cards due**: "All caught up! You have N total cards. Next review: [date]."
- **No cards exist**: "No flashcards yet. Run /flashcards after a coding session to generate your first cards."
- **Script not found**: "The flashcards script wasn't found. The plugin may need to be reinstalled."
